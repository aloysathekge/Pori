## Shipped in this phase

- Manual Project Event creation with one canonical, continuous working
  Session. Opening the Event always resumes that conversation; legacy and
  background conversation rows remain provenance, not competing UI sessions.
- Tenant-scoped Event listing and a trusted templated Event Surface with
  Status, Tasks, complete Trail, Notes, durable Files, and pending Proposals.
- User Task create/update/delete endpoints. Every mutation and its
  `task_changed` Trail entry commit atomically.
- Agent `task_create` and `task_update` product tools with the same Event and
  tenant boundaries, direct Working-state semantics, and atomic Trail writes.
- A product-agnostic async-tool seam in the kernel; existing synchronous tools
  retain their original execution path.
- A read-only Today aggregation grouped by Event with Life first: pending
  decisions, committed Proposals and Trail activity from the last 24 hours,
  and open Tasks.
- Today is the authenticated landing page. The desktop shell now uses an
  Event rail, a central lifetime conversation, and a collapsible context pane
  for Tasks, decisions, Files, and Trail. Users can leave and reopen an Event
  without changing conversations.
- `events.primary_conversation_id` plus a backfill migration makes the product
  choice durable. New Events receive their canonical conversation atomically.
- The Windows `make stop` recipe now passes PowerShell variables correctly.
